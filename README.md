# Snowball - OKX Adaptive Grid Trading Agent

OKX 거래소에서 동작하는 적응형 그리드 트레이딩 봇이다.
시장 변동성을 실시간 분석하여 그리드 간격을 자동 조절하고,
위험 상황에서는 **2명의 AI 에이전트(운영자 + 비판자)가 비대칭 합의**로 대응한다.

## 작동 방식

**이벤트 기반 아키텍처** — 5분마다 모니터링하되, 에이전트는 이벤트 발생 시에만 호출한다:

```
매 5분 (모니터링)
  → 시세/지표 수집 → 텔레그램 리포트
  → 이벤트 감지? → NO → MAINTAIN (봇 그대로)
                  → YES → 멀티 에이전트 의견 → 코드 합의/STOP 가드 → 액션 실행
```

> **그리드봇 헌법 — 모든 LLM 판단의 전제** ([상세](docs/incident-2026-04-26-bot-stop-analysis.md)):
> ① 메커니즘: 변동성으로 돈 번다, 횡보가 최적 / ② 디폴트는 MAINTAIN, 변경은 입증 책임 / ③ STOP은 시스템적 위험 한정 / ④ 정지의 비용은 일일 기회비용 영구 손실

### 에이전트 호출 이벤트

평소에는 모니터링만 하고, 아래 이벤트 발생 시에만 에이전트에게 판단을 요청한다:

| 이벤트 | 조건 | 설명 |
|--------|------|------|
| 높은 리스크 | 리스크 스코어 `LLM_TRIGGER_SCORE` 이상 | 시장 상황 종합 판단 |
| 변동성 급등 + 추세 | ATR이 평균의 3배 이상 + ADX 25+ | 단발성 spike가 아닌 추세성 변동만 판단 |
| 거래량 폭발 + 추세 | 거래량 5배+ + ADX 25+ | 급변이 추세를 동반할 때만 판단 |
| 그리드 이탈 | 가격이 범위 밖으로 이탈 24시간+ | 재배치 여부 판단 (48시간+ 초과 시 LLM 건너뛰고 **강제 SHIFT**) |

그리드 상/하단 80% 접근은 더 이상 에이전트 호출 사유가 아니다. 그 지점은 위험이 아니라 체결 자리이므로 모니터링만 하고, 실제 범위 이탈은 별도 이탈 타이머가 처리한다.

이벤트가 없으면 LLM 호출 비용 **$0**.

### 에이전트 가능 액션

| 액션 | 설명 | 비용 |
|------|------|------|
| **MAINTAIN** | 현재 그리드 유지 (디폴트, 모호하면 무조건 이걸로) | $0 |
| **WIDEN** | 그리드 간격 확대 재시작 | 수수료 발생 |
| **SHIFT_UP** | 그리드 위로 이동 | 수수료 발생 |
| **SHIFT_DOWN** | 그리드 아래로 이동 | 수수료 발생 |
| **STOP** | 봇 종료 + 자산 청산 (시스템적 시장 붕괴 한정) | 수수료 + 일일 기회비용 영구 손실 |

> ~~PAUSE/REDUCE~~ — OKX 그리드봇은 일시정지 API가 없어서 봇 중지→재시작이 필요했고,
> 매번 수수료가 발생해서 제거했다. 봇은 항상 돌리고, 진짜 위험하면 STOP으로 청산한다.

> **STOP은 거의 발동되지 않는다.** 4/26 사고 ([incident report](docs/incident-2026-04-26-bot-stop-analysis.md)) 이후
> STOP 보호 장치가 추가되어, 2명 만장일치 + 전원 신뢰도 9/10+ + 사유에 시스템 위험(거래소 장애, 폭락, 영구 추세 전환)이 명시될 때만 채택된다.
> 그 외엔 SHIFT/WIDEN/MAINTAIN으로 자동 강등된다.

### 멀티 에이전트 합의 시스템 (2 체제)

이벤트 발생 시 **2명의 에이전트**가 **그리드봇 운영자 관점**으로 독립 분석 후 코드 조율자가 합의를 도출한다.
4/26 사고 분석 결과 ([incident report](docs/incident-2026-04-26-bot-stop-analysis.md))
4명 트레이더 페르소나 합의는 "보수적 합의 = STOP"으로 끌려가는 편향이 있어 2 체제로 재설계했다.

모든 에이전트는 시스템 프롬프트 최상단에 **그리드봇 헌법**을 공유한다:

```
【그리드봇 운영 헌법】
1. 메커니즘: 변동성으로 돈 번다. 횡보가 최적 환경. 경계 근처는 "체결 자리"이지 위험 아님.
2. 디폴트: MAINTAIN. 액션 변경의 입증 책임은 변경 쪽에 있음.
3. STOP: 거의 없음. 시스템적 시장 붕괴(±20% 폭락, 거래소 장애, ADX 35+ 영구 추세)에 한정.
4. 정지의 비용: 자본 보전이 아니라 자본 효율 손실 (일일 기회비용 영구 발생).
```

| 에이전트 | 역할 | 분석 초점 |
|---------|------|----------|
| **운영자 (Operator)** | 액션을 제안 | 그리드 작동 평가, ATR/RSI/BB/거래량/EMA/ADX 종합 |
| **비판자 (Exit Critic)** | 액션 변경에 반대 (변호인) | "그냥 두면?" 항상 자문, MAINTAIN 강하게 옹호 |
| 코드 조율자 | 합의 도출 + STOP 가드 | LLM 호출 없음. 변경에는 만장일치 필요, MAINTAIN은 한 명이면 충분 |

**의도된 비대칭 합의:** 변경하려면 두 명을 모두 설득해야 하지만, 유지는 한 명만으로 충분하다. 그리드봇 헌법 ②(MAINTAIN 디폴트)를 구조에 박았다.

**합의 규칙 (2 체제):**
- 두 명 모두 같은 NON-MAINTAIN 액션 동의 + 평균 신뢰도 7 초과 → 채택
- **한 명이라도 MAINTAIN을 답함 → MAINTAIN** (비판자가 반대하면 변경 안 함)
- 둘이 서로 다른 NON-MAINTAIN 액션을 답함 → MAINTAIN (의견 분산)
- 두 의견 신뢰도 평균 7 이하 → MAINTAIN (확신 부족)
- 두 에이전트 동등. 어느 한 쪽에 가중치 없음
- **STOP 채택 조건 (3가지 모두 만족):**
  1. 두 명 만장일치 STOP
  2. 두 명 모두 STOP 신뢰도 9/10 이상 (2명 체제라 더 보수적)
  3. 사유에 시스템 위험 명시 (거래소 장애, ±20% 폭락, ADX 35+ 영구 추세 등)
  - 미충족 시 SHIFT/WIDEN/MAINTAIN으로 코드 레벨 자동 강등
- 그리드 이탈 24h+ + 두 명 동일 NON-MAINTAIN 동의 → 그 액션 채택
- 이탈 48h+ + ADX≥20 + 수수료 가드 통과 → WIDEN 적극 채택
- 1시간 내 재시작 2회+ → WIDEN/SHIFT 금지 (MAINTAIN 강제)
- 예상 재시작 수수료 > 당일 실현 수익의 50% → MAINTAIN 강제
- `MULTI_AGENT_MODE=false`로 끄면 단일 LLM 모드로 폴백

**비용 영향:** 4 체제(5 호출/판단) → 2 체제(2 호출/판단)로 LLM 비용 약 **60% 감소**. 조율자는 LLM이 아니라 코드 규칙이다.

**LLM 컨텍스트 주입 (그리드봇 헌법 ④):**
매 호출마다 봇 운영 현황을 숫자로 주입해서 LLM이 "정지의 비용"을 인식하게 한다:
```
=== 봇 운영 현황 (정지의 기회비용 인식용) ===
누적 실현 손익: +656.49 USDT
운영 기간(추정): 21.0일
일평균 실현 손익: +31.26 USDT/일
⚡ 정지 시 일일 기회비용: ~31.26 USDT/일
```

## 실행

```bash
cd src
pip install -r requirements.txt
python main_agent.py
```

처음 실행하면 **초기 설정 위자드**가 자동으로 시작된다:

1. OKX API 설정 (Key, Secret, Passphrase, Demo/Live)
2. LLM 설정 (Provider, API Key, Model)
3. 거래 설정 (심볼, 총 예산 → AI가 그리드 예산/범위/개수 자동 추천)
4. 텔레그램 알림 (선택, Chat ID 자동 감지)

설정 완료 후 메인 메뉴가 뜬다:

### 설정 메뉴

`⚙️ 설정`에서 각 항목을 개별로 설정한다:

```
? 설정 항목 (↑↓ 이동, Enter 선택)
 ❯ ❌ OKX API
   ❌ 거래 설정
   ❌ LLM 설정
   ⬜ 텔레그램 알림
   ⬜ 고급 설정
   ──────────────
   ← 뒤로
```

- **방향키(↑↓)** 로 이동, **Enter** 로 선택
- **API 키**는 비밀번호 마스킹(`****`)으로 입력
- **LLM 제공자/모델** 등은 방향키로 골라서 선택
- **텔레그램 알림 상태**는 **Space**로 중복 선택 가능

```
? 알림 받을 상태 (↑↓ 이동, Space 선택/해제, Enter 확인)
 ❯ ◉ CAUTION (주의)
   ◉ WARNING (경고)
   ◉ EMERGENCY (긴급)
```

설정은 `.env` 파일에 저장된다. `Ctrl+C`로 종료한다.

### 기존 그리드봇 자동 연결

에이전트 시작 시 OKX에서 이미 돌아가는 그리드봇이 있으면 **자동으로 감지하고 동기화**한다:

```
════════════════════════════════════════════════════════════
  ✅ 기존 그리드봇에 연결되었습니다
════════════════════════════════════════════════════════════
  봇 ID    : **********
  상태     : running
  범위     : 2,130.00 ~ 2,210.00
  그리드   : 20개 (arithmetic)
  투자금   : 400.00 USDT
  현재 손익 : +419.49 USDT
────────────────────────────────────────────────────────────
  기존 설정에 맞춰 에이전트를 시작합니다.
```

없으면 새 그리드봇을 생성한다. 잔고 부족 시 원인과 해결 방법을 안내한다.

### 텔레그램 알림

- **이벤트 기반 발송이 디폴트** — `NOTIFY_TICK_REPORTS=false`(기본값) 시 매 틱 요약은 생략하고 아래 이벤트에만 알림. 알림 폭주 방지.
- **이벤트 알림** (항상 발송):
  - LLM 멀티 에이전트 합의 결과
  - WIDEN / SHIFT_UP / SHIFT_DOWN / STOP 액션 실행
  - 매수/매도 체결
  - 그리드 이탈 첫 감지 / 범위 복귀 / 하드 타임아웃 강제 재배치
  - 상태 변화 (`NOTIFY_ON_STATES` 지정 — 기본 CAUTION/WARNING/EMERGENCY)
  - 손절 청산 / 긴급 청산
  - 일일 리포트 (`DAILY_REPORT_HOUR` 시간에 PnL + 체결 요약)
- **EMERGENCY 시** 10초 간격으로 4회 반복 긴급 알림 발송 (`NOTIFY_TICK_REPORTS` 무관 항상)
- **매 틱 요약 (옵션)** — `NOTIFY_TICK_REPORTS=true`로 켜면 5분마다 가격/지표/봇 상태 요약 발송
- **설정 시 자동 검증** — Bot Token 유효성 확인 + Chat ID 자동 감지 + 테스트 메시지 발송
- **봇 라벨 자동 prefix** — 멀티봇 환경에서 어느 봇 알림인지 즉시 식별 가능
- **봇 리스트 footer (옵션)** — `NOTIFY_INCLUDE_BOT_LIST=true`(기본값) 시 모든 알림 끝에 OKX 활성 그리드봇 리스트 자동 첨부 (60초 캐시):

```
─── 운영 중 봇 ───
🟢 1700-2100 (12 RT, 📈 +45 USDT)
🟢 2000-2500 (3 RT, 📈 +18 USDT)
```

### 멀티봇 환경 알림 핸들링

여러 그리드봇을 같은 텔레그램 채널로 운영해도 알림 prefix로 깔끔하게 구분된다:

```
[ETH 1700-2100] 📈 그리드 상향 시프트 | ETH-USDT
[ETH 2000-2400] 🤖 멀티 에이전트 합의 [ETH 2000-2400]
[ETH 2000-2400] ✅ 그리드 범위 복귀 | ETH-USDT
[ETH 1700-2100] 🛡️ STOP 후보 → MAINTAIN 강등됨
```

**라벨 자동 생성 (`Notifier.bot_label`):**
1. `config.BOT_LABEL` 환경변수 (사용자 별칭) — 설정되어 있으면 우선
2. 런타임 그리드 범위 — `{base} {lower}-{upper}` (예: `ETH 2000-2400`)
3. WIDEN/SHIFT로 그리드 범위가 바뀌면 라벨도 즉시 갱신되어 다음 알림부터 새 prefix 적용

**LLM 컨텍스트에도 봇 식별:**
멀티 에이전트 합의 시 `_build_market_context`에 `[봇: ETH 2000-2400]` 헤더가 박혀, LLM이 어느 봇 판단인지 인식한 채로 응답한다.

**N봇 동시 운영 (현재 권장 구성):**
각 봇마다 별도 main_agent 프로세스를 띄우고, `.env`로 `SYMBOL`/`GRID_LOWER`/`GRID_UPPER`/`BOT_LABEL`만 다르게 설정한다. 같은 텔레그램 채널을 공유해도 라벨로 구분된다.

> 단일 프로세스 N봇 오케스트레이터(`MultiBotOrchestrator`)는 향후 마일스톤이다.
> 현재는 봇별 독립 프로세스 + 알림 라벨 prefix 조합으로 운영한다.

### 대기 시간 시각화

다음 틱까지 프로그레스 바로 대기 시간을 시각적으로 표시한다:

```
  ⏳ ████████████████░░░░░░░░░░░░░░░░░░░░░░░░  42.5% (다음 틱까지 1:09)
```

### 그리드 이탈 자동 대응

가격이 그리드 범위를 이탈하면 단계별로 대응한다:

| 이탈 기간 | 동작 |
|-----------|------|
| 0 ~ 24시간 | 알림만 발송, 가격 복귀 대기 |
| 24시간+ | 멀티 에이전트에 재배치 판단 요청 (이탈 기간 컨텍스트 전달) |
| **48시간+** | **LLM 건너뛰고 방향별 강제 SHIFT** (수수료 가드는 여전히 적용) |

LLM 프롬프트에 이탈 기간별 가이드라인이 주입되어, 24h 이상 이탈이면 복귀 대기보다 재배치를 우선 고려한다. 기회비용(현금 놀림)이 재시작 수수료를 넘으면 재배치가 합리적이기 때문이다.

### 상태 영속성 (재시작 안전)

에이전트 재시작 / 업데이트 시 이탈 타이머와 당일 카운터를 이어받는다:

- 매 틱 종료 시 `src/agent_state.json`에 상태 저장 (`.tmp` → `os.replace`로 원자적 쓰기)
- 시작 시 자동 복원: 이탈 타이머 3종 / 재시작 기록 / 포지션 / 누적 수수료 / 당일 카운터
- 저장 날짜 ≠ 오늘이면 `daily_*` 카운터만 초기화, 나머지는 유지
- 심볼 불일치 시 복원 건너뜀 (다른 봇 상태 혼입 방지)
- 저장 실패해도 루프 계속, 에러 로그만 남는다
- `STATE_FILE=""` 로 비활성화 가능

업데이트하고 재시작해도 "4일째 이탈 중이었는데 타이머가 0부터 다시 시작" 같은 문제가 없다.

### 비용 인식 시스템 (CostGuard)

[Claude Code 아키텍처](https://blog.aldente0630.com/insights/claude-code-architecture-analysis/)에서 영감받은 비용 최적화 시스템:

**에러 복구 캐스케이드** — 항상 무료부터 시도한다:

```
LLM 호출 실패 시:
  Level 0 (무료) → 캐시 재사용 / 마지막 액션 반복
  Level 1 (무료) → 보수적 MAINTAIN 폴백 (score만으로 WIDEN/STOP 금지)
  Level 2 (저비용) → 단일 LLM 호출 (멀티 에이전트 대신)
  Level 3 (고비용) → 풀 멀티 에이전트 재시도
```

**서킷 브레이커** — 5회 연속 실패 시 5분간 LLM 호출 차단한다:

```
CLOSED ──5회 실패──→ OPEN ──5분 쿨다운──→ HALF_OPEN ──성공──→ CLOSED
```

**감소 수익 감지** — 3회 연속 동일 판단 시 API 호출 스킵한다:

```
MAINTAIN → MAINTAIN → MAINTAIN → 스킵! (비용 $0)
시장 변동 감지 (스코어 5점+ 변화) → 자동 리셋 → 다시 호출
```

**응답 캐시** — 동일 시장 상황에서 중복 호출 방지한다:

```
시장 상태 해시 (스코어 5점 양자화 + 상태 + 추세)
→ 5분 TTL 내 동일 조건 → 캐시 결과 반환 (API 호출 0)
```

**일일 예산 한도** — 일일 $5 초과 시 자동 MAINTAIN 폴백. score만으로 재시작/정지를 실행하지 않는다.

- 손절 기준을 stale `entry_price`가 아니라 실제 평균단가와 총손익(실현+미실현) 기준으로 변경

- WIDEN이 현재 그리드보다 좁아지지 않게 보장하고, 기존 그리드 개수/모드를 유지
- 당일 실현수익이 없고 총손익이 음수인 WIDEN은 보류
- 2인 에이전트 합의를 코드 규칙으로 강제: 한 명이라도 MAINTAIN이면 변경 금지, 조율자 LLM 제거
- 그리드 80% 접근은 LLM 호출 사유에서 제거. 실제 이탈은 24h 대기 후 판단
- LLM 실패/예산 초과 시 score만 보고 WIDEN/STOP하지 않고 MAINTAIN으로 폴백
